# NFT1155 Contract Design (Base Sepolia)

## Overview
NFT1155 dung de mint nhieu ban sao cho mot noi dung digital item. Khac voi ERC721, ERC1155 cho phep supply > 1 va nhieu nguoi co the so huu cung tokenId. Contract hien tai da co totalSupply, whitelist, mint window va metadata theo token.

Muc tieu tai lieu nay:
- Chuan hoa supply management
- Chuan hoa single mint va batch mint
- Dam bao backend co the dong bo inventory voi marketplace

## Contract Responsibilities
- Mint tokenId moi voi so luong amount xac dinh
- Theo doi tong cung moi tokenId
- Quan ly URI metadata moi tokenId
- Ho tro mint nhieu token trong 1 giao dich (batch mint)
- Emit event de backend cap nhat listing va ton kho

## State Variables
- currentTokenId: bo dem token id ERC1155
- mintEndTime: han mint
- totalSupply[tokenId]: tong cung da mint cho tokenId
- whitelist[address]: dia chi duoc phep mint
- tokenURIs[tokenId]: URI metadata rieng cho token
- contentToTokenId[bytes32]: optional map content key -> tokenId neu can unique theo content
- maxSupplyPerToken: gioi han supply de tranh mint qua lon

## Core Functions (description only)
- setWhitelist(user, allowed)
  - Owner cap/thu hoi quyen mint
- setMintEndTime(newTime)
  - Owner cap nhat han mint
- setMaxSupplyPerToken(newMax)
  - Owner dat gioi han amount
- mint(to, amount, tokenURI, contentKey)
  - Validate mint window, whitelist, amount > 0, amount <= max
  - Tao tokenId moi, set totalSupply, set URI, mint amount cho to
  - Co the map content -> tokenId neu muon enforce mot token class cho moi content
- mintBatch(to, amounts[], tokenURIs[], contentKeys[])
  - Validate do dai mang khop nhau
  - Loop tao nhieu tokenId lien tiep, set supply va URI tung token
  - Mint batch de tiet kiem gas
- uri(tokenId)
  - Tra metadata URI

## Events
- WhitelistUpdated(user, allowed, updatedBy)
- MintWindowUpdated(oldTime, newTime)
- MaxSupplyUpdated(oldMax, newMax)
- TokenMinted1155(tokenId, to, amount, contentKey, tokenURI, caller)
- BatchMinted1155(startTokenId, count, to, caller)

## Access Control
- Owner:
  - Quan tri whitelist, mint window, max supply
- Whitelisted minter:
  - Duoc mint va mintBatch
- User mua ban:
  - Transfer token qua safeTransferFrom/safeBatchTransferFrom thong qua marketplace flow

Khuyen nghi production:
- Tach MINTER_ROLE va OPERATOR_ROLE neu can phan quyen chi tiet

## Flow Diagrams (text-based)
### Single Mint ERC1155 Flow
1. User chon ERC1155 + amount
2. Frontend gui request den backend
3. Backend validate amount va metadata
4. Backend goi mint(to, amount, tokenURI, contentKey)
5. Contract mint tokenId moi va emit TokenMinted1155
6. Worker cap nhat bang nfts + nft_listings trong Supabase

### Batch Mint ERC1155 Flow
1. Creator muon tao nhieu asset trong 1 lan
2. Frontend gui danh sach metadata + amount
3. Backend goi mintBatch
4. Contract tao nhieu tokenId, emit BatchMinted1155
5. Worker tao nhieu listing records theo tokenId vua sinh

## Edge Cases
- amount = 0:
  - Revert bat buoc
- amount qua lon gay rui ro inventory/gas:
  - Gioi han boi maxSupplyPerToken
- Batch co mang do dai lech nhau:
  - Revert de tranh mapping sai
- Mint on-chain ok, listing tao fail:
  - Retry job dua tren event idempotency key
- Metadata trung lap:
  - Cho phep neu business chap nhan; neu khong, backend validate truoc mint

## Security Considerations
- Validate amount o ca backend va contract
- Han che batch qua lon de tranh gas spike va DoS logic
- Khong cho user client goi mint truc tiep neu model can kiem soat chat; uu tien backend minter wallet
- Event parser phai xu ly duplicate delivery va chain reorg
- Neu su dung contentToTokenId, phai xac dinh ro co enforce unique hay khong de tranh xung dot voi logic ERC1155 multi-edition
